解决Git切换用户push报错403问题

发表于2021-02-10|更新于2021-02-10|工具使用

向自己的Git仓库提交代码时，报错：push的时候报错 403：xxxxxx. 意思为当前存在两个账户，我自己的账户没有提交代码的权限。

使用全局信息配置好像不大行

git config --global usr.name ""

git config --global usr.emial

但本地还是没有权限

找到了解决办法，在Windows设置 —> 控制面板里找到用户凭据，管理Windows凭据里修改git的信息。
